How to use a MIDI keyboard in a browser DAW (Web MIDI)

By Tongkit Leong · June 21, 2026 · 7 min read

To record a MIDI keyboard in your browser, plug it in via USB, open Chrome or Edge, allow MIDI access when the browser asks, and select the device in Tracks — the Web MIDI API handles the connection without any drivers or software install on your end.

What the Web MIDI API actually does

The Web MIDI API is a browser feature that lets a web page read MIDI data from connected hardware — keyboards, drum pads, controllers, interfaces — directly through a USB connection. It works the same way a desktop DAW reads MIDI, except the permission gate is the browser rather than a driver installer.

Chrome and Edge support Web MIDI natively as of their current versions. Firefox and Safari do not include it by default. If you are using Firefox or Safari, you will need to switch to Chrome or Edge for MIDI recording to work.

On the hardware side, any keyboard or controller with a standard USB MIDI connection works. Bluetooth MIDI is not yet covered by the Web MIDI API in most browsers, so a physical USB cable is the reliable option.

What you need

  • A MIDI keyboard or controller with a USB output (most modern keyboards have one)
  • A USB cable connecting the keyboard to your computer
  • Google Chrome or Microsoft Edge (any reasonably recent version)
  • The Tracks studio open in a browser tab

You do not need to install any drivers on Windows or Mac for class-compliant USB MIDI devices — operating systems handle them automatically. If your keyboard requires its own driver, install that first, then connect.

Step 1: Connect the keyboard and open the studio

Plug the keyboard into a free USB port. Most keyboards power on automatically when connected to a computer via USB; check your keyboard's manual if it does not. Open Chrome or Edge and navigate to the Tracks studio.

It does not matter whether you connect the keyboard before or after opening the browser tab, but connecting before the page loads makes device detection slightly more reliable across different operating systems.

Step 2: Allow MIDI access

The first time Tracks tries to read from a MIDI device, Chrome or Edge will show a permission prompt in the browser toolbar asking whether to allow MIDI device access. Click Allow.

If the prompt disappeared before you responded, or if you previously clicked Block, fix it this way:

  1. Click the lock icon (or the tune icon in some Chrome versions) to the left of the address bar.
  2. Find MIDI devices in the permissions list.
  3. Change the setting from Block to Allow.
  4. Reload the page.

You only need to do this once per browser profile. Chrome and Edge remember the permission for the site.

Step 3: Select your MIDI device in Tracks

Inside the Tracks studio, open the MIDI settings panel. Your connected keyboard should appear in the input device list by its model name — for example, "Arturia KeyStep 37" or "KORG microKEY2". Select it.

If the device does not appear, try these steps in order:

  • Unplug and re-plug the USB cable, then reload the page.
  • Confirm that Chrome or Edge has MIDI access permission (Step 2 above).
  • On Windows, open Device Manager and check that the keyboard appears under Sound, video and game controllers without an error icon.
  • On Mac, open the Audio MIDI Setup app (found in /Applications/Utilities/) and confirm the keyboard appears in the MIDI Studio view.

Step 4: Play an instrument live

Add an instrument track in the Tracks timeline. You can choose from the subtractive synth, the FM synth, the 808 or 909 drum machine, or a SoundFont (.sf2) player. With the track selected and MIDI input routed to it, play a key on your keyboard — the instrument should respond in real time through your computer's audio output.

If you hear a delay between pressing a key and hearing the sound, that is MIDI or audio latency. Lower the audio buffer size in Tracks' audio settings if the option is available, and close other browser tabs or applications consuming CPU. Most setups get latency below 20 ms, which is imperceptible for most players.

Step 5: Record your performance

Enable the record arm button on the instrument track you want to record onto. Then press the main Record button in the transport bar. Tracks plays a count-in bar — one full bar at the project tempo — before recording starts, so you have time to get ready. Perform your part, then press Stop.

The recorded notes appear as a MIDI clip on the timeline. Every note's pitch, velocity (how hard you pressed the key), and duration is captured exactly. Velocity matters for expressive playing: lighter touches produce quieter, softer notes on most instruments, while harder presses produce more aggressive, louder attacks.

Your project data — including the MIDI recording — stays entirely in your browser and is never uploaded anywhere. See the beginner's guide to making a song for more detail on how the arrangement view and transport controls work together.

Step 6: Edit in the piano roll

Double-click the recorded MIDI clip to open it in the piano roll. This is where you fix mistakes, adjust timing, and shape the feel of the part without re-recording anything.

Correcting notes

If you hit the wrong key at any point, click the incorrect note to select it and delete it with the Delete key, then draw the correct note in its place with the pencil tool. To move a note to a different pitch, drag it up or down. To move it in time, drag it left or right.

Quantizing timing

If your timing was slightly off, select all notes (Ctrl+A or Cmd+A) and apply quantize. Choose the quantize resolution — 1/8 note is common for melodic lines, 1/16 for faster passages. Quantize snaps each note's start to the nearest grid line at that resolution. For a more human feel, use swing quantize rather than straight quantize, which nudges off-beat notes slightly later rather than aligning them to a rigid grid.

Adjusting velocities

The piano roll in Tracks shows note velocity as bar height at the bottom of the editor. Click and drag individual velocity bars up or down to make specific notes louder or quieter. This is useful for accenting certain beats or calming down notes that came out too aggressive. Selecting multiple notes and dragging scales all of them proportionally.

Step 7: Record multiple takes

Tracks supports take comping — recording multiple passes of the same section and then choosing the best notes from each pass to build a final composite performance. Enable take comping in the transport, then record the section as many times as you want. Each pass is stored as a separate take within the same clip.

After recording, open the clip and switch between takes to hear each one in context. You can then cut between takes in the piano roll, keeping the best bar from take 1, the best run from take 2, and so on. The result sounds like one smooth performance.

This workflow works particularly well for solos, melodic leads, and anything where feel and expression matter more than technical precision. Record five or six passes without stopping, then spend a few minutes in the piano roll assembling the best version. It is faster than trying to nail everything in a single take.

Using your MIDI keyboard for more than melodic parts

A MIDI keyboard connected to Tracks can play any instrument — not just melodic synths. Route it to the 808 or 909 drum machine and each key triggers a different drum sound. Route it to the 16-pad sampler and each key plays a different sample. This makes the keyboard a versatile performance tool for building beats and layering sounds, not just writing melodies.

You can also use the keyboard while mixing. A MIDI controller with knobs or faders can control track volumes, EQ parameters, and send levels if you map its controls to those parameters in Tracks. Keyboard keys map to notes in the piano roll by default, but knobs and faders on the same device can control almost any mixer parameter when MIDI learn is active.

For tips on shaping your recorded parts with EQ and compression after the performance is in, see How to mix a track: EQ, compression, and reverb basics.

Frequently asked questions

Does Web MIDI work in Safari or Firefox?

Not by default. Safari does not include Web MIDI support. Firefox does not support it natively either, though a third-party browser extension exists that adds partial support. For reliable MIDI recording with no extra steps, use Chrome or Edge.

Do I need to install drivers for my MIDI keyboard?

Usually not. Most USB MIDI keyboards are class-compliant, meaning Windows and macOS recognize and connect to them automatically with no additional software. Some keyboards — particularly those with advanced audio features or proprietary firmware — require the manufacturer's driver. Check your keyboard's documentation; if a driver is needed, install it before connecting the keyboard and opening the browser.

Can I use a MIDI controller (not a keyboard) with Tracks?

Yes. Drum pads, pad controllers, breath controllers, and any other MIDI-over-USB device that sends standard MIDI messages work the same way. Drum pads send note-on and note-off messages just like keyboard keys do. The note number determines which pitch or drum sound is triggered, and the velocity determines the volume.

Can I record MIDI from a keyboard on one track while other tracks play back?

Yes. Arm only the track you want to record onto, then start recording. All other tracks in the Tracks timeline play back normally during the recording pass, so you can hear your drums, bass, and chord tracks while you perform the new part on top of them.

Does my MIDI recording stay saved if I close the browser tab?

Your project is saved in your browser's local storage and is not uploaded to any server. It will still be there when you reopen the tab in the same browser on the same device. However, local browser storage can be cleared by certain browser settings or privacy tools. To be safe, export your project as a .tracks.json file to your downloads folder after any session you want to keep. You can reload it by importing it back into the studio.

What if my keyboard is connected but Tracks shows no MIDI input?

First confirm that Chrome or Edge has MIDI device permission for the site (check the lock icon in the address bar). Then unplug and re-plug the keyboard and reload the page. On Windows, check Device Manager to confirm the keyboard appears without errors. On Mac, open Audio MIDI Setup and verify the keyboard shows up in the MIDI Studio view. If it appears there but not in Tracks, a page reload after the device is recognized usually resolves it.
